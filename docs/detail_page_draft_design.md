# 매실 스튜디오 — 상세페이지 초안 생성 시스템 설계서

> 품질 개선을 위한 리뷰용 문서. GPT/Gemini와 함께 검토.

---

## 1. 전체 플로우

```
사용자 입력
    ↓
[Phase 1] 3가지 타입 기획 (섹션 구조만) — Celery 비동기
    ↓ Claude claude-sonnet-4-6, max_tokens=4096
    ↓
사용자가 3개 중 1개 타입 선택
    ↓
[Phase 2] 선택 타입 카피 생성 — Celery 비동기
    ↓ Claude claude-sonnet-4-6, max_tokens=4096
    ↓
사용자가 원하는 섹션만 이미지 생성 (FLUX, 50P/장)
    ↓
PNG / PDF 내보내기
```

---

## 2. 사용자 입력값

| 필드 | 필수 | 설명 |
|------|------|------|
| brand_id | O | 브랜드 프로필 (tone, 타겟, 소개 등) |
| product_name | O | 상품명 |
| features | O | 핵심 특징 (쉼표 구분 또는 자유 텍스트) |
| target_customer | - | 타겟 고객 (없으면 브랜드 프로필 기준) |
| price_range | - | 가격대 |
| differentiator | - | 경쟁 대비 차별점 |

> **기존 상품 불러오기**: `products` 테이블에 저장된 상품 선택 시 자동 채움.  
> features 없으면 description fallback.

## 3. Phase 1 — 섹션 구조 기획

### 3-1. System Prompt 핵심
```
당신은 대한민국 최고 수준의 온라인 커머스 마케팅 전략가.
쿠팡·네이버스마트스토어 전환율 최적화 전문가.

핵심 원칙:
- '우리 제품이 좋다' X → '고객의 고민을 우리가 해결한다' O
- 고객은 스크롤하며 자신의 이야기를 찾는다 — 첫 3초에 공감
- 각 섹션 = 구매 심리 여정의 한 단계
```

### 3-2. 3가지 타입 전략

#### 공감·문제해결형 (PAS)
- **프레임워크**: Problem → Agitation → Solution
- **핵심 논리**: 고객 불편을 먼저 말해줌 → "이 브랜드는 나를 안다" 신뢰 형성
- **구매 여정**:
  1. 공감 훅 → "혹시 나만 이런 건가?"
  2. 문제 심화 → "맞아, 그래서 계속 힘들었구나"
  3. 해결책 등장 → "어떻게 다른 거지?"
  4. 납득 근거 → "믿어도 될까?"
  5. 사회적 증거 → "나랑 똑같은 사람들이 해결했구나"
  6. 결심 유도 → "혹시 후회하면 어떡하지?" 반론 제거

#### 스토리·라이프스타일형 (BAB)
- **프레임워크**: Before → After → Bridge
- **핵심 논리**: 제품보다 삶의 모습을 먼저 팔고, 제품은 그 삶으로 가는 다리
- **구매 여정**:
  1. 비전 제시 → "맞아, 나도 이렇게 살고 싶어"
  2. 현실 대비 → "맞아, 현실은 이렇지…"
  3. 브랜드 철학 → "이 브랜드가 나의 가치관과 맞네"
  4. 경험/제품 → "실제로 어떻게 달라지는 거지?"
  5. 변화 스토리 → "나도 저렇게 될 수 있겠다"
  6. 초대 → "나도 해볼까?"

#### 데이터·전문가형 (ACCA)
- **프레임워크**: Awareness → Comprehension → Conviction → Action
- **핵심 논리**: 수치·인증·비교로 이성적 납득 → 신뢰
- **구매 여정**:
  1. 임팩트 수치 → "오, 이게 사실이야?"
  2. 문제 정의 → "이런 문제가 있는 줄 몰랐네"
  3. 기술 차별화 → "경쟁 제품이랑 뭐가 다르지?"
  4. 전문가 검증 → "누군가 검증했나?"
  5. 실증 데이터 → "실제로 효과 있나?"
  6. 선택 정리 → "가성비 있나? 후회 안 하나?"

### 3-3. Phase 1 출력 JSON 구조
```json
{
  "plans": [
    {
      "type_name": "공감·문제해결형",
      "hook": "핵심 전략 한 줄 (40자 이내)",
      "appeal_analysis": {
        "target_customer": "구체적 상황 묘사 (직업·나이 말고 상황으로)",
        "core_pain": "고객이 직접 할 말투로",
        "buy_trigger": "구매 버튼을 누르게 만드는 심리적 마지막 한 마디",
        "appeal_points": ["포인트1", "포인트2", "포인트3"]
      },
      "sections": [
        {"no": 1, "name": "고객 감정/의문 중심 섹션명 (12자 이내)", "purpose": "고객 머릿속에 심어야 할 생각 (35자 이내)"}
        // ... 6개
      ]
    }
    // ... 3개 타입
  ]
}
```

## 4. Phase 2 — 카피 생성

### 4-1. 입력
- Phase 1에서 선택된 플랜 전체 (type_name, hook, appeal_analysis, sections)
- 원본 input_data (상품 정보)
- 브랜드 컨텍스트

### 4-2. 섹션 × 구매 여정 매핑
각 섹션(6개)을 선택한 타입의 구매 여정 단계(6개)와 순서 매칭:
- 섹션 역할 (j_role)
- 고객의 내면 목소리 (j_psych)
- 카피 각도 (j_angle)
- 섹션 목적 (purpose)

### 4-3. 카피 규칙
```
구조: 헤드라인(1줄) + 본문(2~3줄)

헤드라인 (12자 이내):
  공감형 → 고객 상황/고통을 직접 꼬집는 문장
  스토리형 → 장면·감정이 담긴 감각적 한 줄
  데이터형 → 숫자·사실로 시작하는 임팩트 문장

본문 (각 줄 28자 이내):
  줄1: 고객 상황 공감 또는 문제 구체화
  줄2: 제품이 어떻게 해결하는지 (기능 X, 고객이 얻는 결과 O)
  줄3(선택): 수치/성분/후기 또는 다음 섹션으로 이어지는 여운

절대 금지 표현:
  최고, 최상, 최저가, 품질 좋은, 믿을 수 있는, 정성껏 만든,
  특별한, 엄선된, 프리미엄, 합리적인

기능→편익 변환 예시:
  X: "HACCP 인증을 받았습니다"
  O: "아이 입에 들어가는 거, 인증 없는 건 이제 못 사겠더라고요 — HACCP 통과"
```

### 4-4. Phase 2 출력 JSON 구조
```json
{
  "copies": [
    {"no": 1, "copy": "헤드라인\n본문줄1\n본문줄2", "image_prompt": "English FLUX prompt"}
    // ... 6개
  ]
}
```

## 5. Phase 3 — 이미지 생성

- 모델: FLUX (별도 API)
- 비용: 50P / 장
- 섹션별 영문 image_prompt 사용 (Phase 2에서 생성)
- 보조 fallback: `scene_map` — 섹션명/purpose 키워드로 촬영 스타일 결정

```
섹션 감정 → 촬영 스타일:
  공감/스토리 → warm natural light, lifestyle, authentic feel
  증거/데이터 → clean studio, clinical, precise, overhead shot
  감성/결심 → soft bokeh, warm tones, emotional
```

## 6. 현재 품질 평가 (베마마 이유식 사례 기준)

### 잘 되는 것
- 섹션 구조가 PAS 여정을 잘 따름
- 고객 언어(후기 말투)가 살아있음
- 기능→편익 변환 비교적 잘 됨
- 안전성 불안 섹션(HACCP)이 자연스럽게 연결됨

### 아쉬운 것
- 헤드라인이 12자 제한에 걸려 힘이 빠지는 경우
- 마지막 섹션(결심 유도)의 마무리 문장이 약함
- 이미지 프롬프트가 섹션 내용과 100% 매칭되지 않을 때가 있음
- 브로셔 문체가 간헐적으로 섞임 ("손질부터 냄새까지 대신 해드리니" 같은 표현)
- 6개 섹션 전체가 하나의 스토리로 읽히지 않고 섹션별로 끊기는 느낌

## 7. 개선 방향 후보 (검토 요청)

아래 항목들을 GPT/Gemini와 함께 검토하고 싶습니다.

### A. 프롬프트 구조 개선
1. **헤드라인 글자수 제한 완화** — 12자 → 15~18자로 늘려도 되는가?
2. **섹션 수 유연화** — 상품 유형에 따라 6개가 아닌 5~8개를 허용해야 하는가?
3. **스토리 연속성 지시 강화** — 6개 섹션이 이어지는 "드라마 대본" 느낌을 더 명시적으로 요구하는 방법은?
4. **카테고리별 금지/권장 표현 세분화** — 이유식, 뷰티, 식품, 생활용품 등 카테고리마다 다른 언어 패턴이 필요한가?

### B. 입력 품질 개선
5. **features 구조화** — 현재 자유 텍스트 → 성분/기능/인증/수치 등 항목별 입력으로 바꾸면 품질이 올라가는가?
6. **리뷰 데이터 활용** — 실제 고객 리뷰 텍스트를 입력받아 core_pain/buy_trigger 추출에 활용하는 방법은?

### C. 생성 전략 개선
7. **2-pass 생성** — Phase 2를 "초안 생성 → 자기 검토 → 개선" 2단계로 나누면 품질이 올라가는가? (비용/시간 trade-off)
8. **타입 혼합** — 순수 PAS/BAB/ACCA보다 혼합형(예: PAS 구조에 BAB 감성)이 더 효과적인 경우가 있는가?
9. **appeal_analysis를 Phase 2에 더 강하게 반영** — 현재 core_pain/trigger가 프롬프트에 포함되지만 카피에서 희미해지는 경향. 섹션별로 "이 섹션에서 반드시 언급할 포인트"를 명시하는 방식은?

### D. 이미지 품질 개선
10. **image_prompt 구체화** — 현재 분위기 중심 → 상품이 실제로 어떻게 등장하는지 장면 묘사를 더 상세히 지시하면 섹션-이미지 매칭이 개선되는가?
